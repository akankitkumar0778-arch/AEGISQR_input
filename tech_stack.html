Built With the
Right Tools.

AegisQR is built using a focused, modern tech stack — every technology chosen for a specific reason, nothing unnecessary.

Foundation
Base Technologies

The core building blocks that every part of AegisQR is built upon.

HTML
Structure

Provides the semantic structure for every page in AegisQR — from the login screen to the emergency profile view scanned by first responders.

CSS
Styling

Handles all visual design — layout, colours, typography, and responsiveness. Every screen is styled to work cleanly on mobile and desktop alike.

JavaScript
Interactivity

Powers all client-side logic — form validation, dynamic UI updates, QR code generation, and communication with the backend API.

Frontend Framework
React.js

React powers the interactive dashboard and profile management interface.

React.js
UI Framework

Used to build the user dashboard where people manage their medical profiles, update personal information, and download their QR codes. React's component-based architecture keeps the interface fast, modular, and easy to maintain as features grow.

Backend
Node.js + Express.js

The server-side layer that handles all API requests, authentication, and data flow.

Node.js
Runtime

The JavaScript runtime that runs AegisQR's server. Its non-blocking, event-driven model handles many simultaneous QR scan requests efficiently — essential for emergency situations where delays are unacceptable.

Express.js
API Framework

A minimal web framework layered on top of Node.js. It defines and manages all REST API routes — including user registration, login, profile updates, and the public QR scan endpoint that first responders access.

Databases
MongoDB + MySQL

Two databases used together — each suited to the type of data it stores.

MongoDB
Medical Profiles

Stores medical profile data as flexible documents. Since every user's health information is different — varying conditions, varying number of medications — MongoDB's schema-flexible model is a natural fit.

MySQL
User Accounts

Stores structured user account data — email addresses, hashed passwords, account status, and login records. MySQL's relational structure enforces data integrity for the authentication system.

Authentication & Security
JWT + bcrypt

Two focused tools that together handle secure login sessions and password protection.

JWT
Login Sessions

JSON Web Tokens are issued when a user logs in and sent with every subsequent API request. The server verifies the token to confirm the user's identity without needing to query the database on every request. Tokens expire automatically for security.

bcrypt
Password Hashing

Before any password is stored, bcrypt transforms it into a secure hash using a salt and a configurable work factor. Even if the database were ever compromised, the original passwords could not be recovered from the stored hashes.
